论文思路

  • introduction

因为大规模卫星网路的发展特别是已经投入使用的starlink，借助于卫星通信的远距离低时延特性，使得在卫星网络中传输高清，2k, 4kl, 360等视频带来的可能。用户不仅可以像地面网一样体验各种业务，而且可以通过卫星实现远距离低延时通信，而且在山区、荒漠、航海、航空等区域，由于地面网络建设困难等因素无法实现覆盖，用户可以借助卫星网络在这些地区实现像地面网一样的上网需求；同时，卫星融合5G可以在物联网设备以及飞机、轮船、火车、汽车等移动载体用户提供连续不间断的网络连接服务能力。卫星通信还可以借助5G网络高传输速率，以提升低轨星座系统的用户体验。

因为卫星的规模越来越大，卫星的数量越来越多，以及全球互联网用户分布和地面站数量、配置缓慢、流量工程不灵活、服务质量（QoS）细粒度等原因，使得大规模卫星网络的利用率分布不均、在互联网用户多的地区上空卫星网络的繁忙，网络使用率高， 互联网用户少的地区网络使用率低。

需要对流量进行规划，使得网络的整体利用率变高，同时尽最大努力满足多用户的QoS需求。

利用软件定义卫星通信网络 (SDSN) 的软件框架。因为该框架可以实现灵活的卫星网络流量工程和细粒度的QoS保证，保证多用户的QoS需求。

简要介绍自己的网络架构和软件说明。

  2. 算法：对流量进行规划，使得网络的整体利用率变高，同时保证用户的QoS需求。

初步实验简化网络规模，9*9/9/0 altitude=550km inclination= 53

  • 星地网络流量工程—-gs-sate-sat–sat-gs

  1. 星地连接问题：源目的地之间的接入卫星选择相同类型的接入卫星

    1. 一个地面站上方的卫星有两种类型，一种是上升卫星（东北方向移动），一种是下降卫星（东南方向移动）。选择相同类型的卫星可以避免最短路径所需要的跳数。参考论文Analysis of Inter-Satellite Link Paths for LEO Mega-Constellation Networks

    2. 确定了源点汇点卫星，接下来进行数据传输。数据在网络中分为3种类型,分为qos1, qos2, qos3。qos1流量需要高带宽、低丢包率、短时延和低抖动。流量可能是视频会议等应用。 qos2 识别需要实时传输但需要低带宽。例如指令数据。qos3 延迟容忍但需要高带宽的流量。例如文件传输数据，或者数据库查询数据。

    3. 传输模型：不考虑如何为地面站选择足够的接入卫星，而是假设要进行传输数据的两颗卫星已给出。

      1. 求出源点到汇点的所有路径，PATH={path1, path2, path3}, 对所有路径进行量化，
        1. path特征， 从球形矩阵源点到汇点的有向图路径问题，（类似于曼哈顿街网络）,即，变成有向无环图（DAG），同时所有 SDP （shortest distance path）都属于MHP （minimum hop path）集合。约束：具有小倾角（小于 68°）或大相位偏移的星座。starlink, inclination=53。满足条件。from-LEO Satellite Networks: When Do All Shortest Distance Paths Belong to Minimum Hop Path Set? letter
      2. 为源与目的地卫星寻找路由(球形矩阵网络带权图）—问题转化为路径规划/图搜索问题：DFS/BFS/Dijkstra/A*
        1. Dijkstra’s Shortest Path （BFS在带权图的扩展) 在卫星网络中即为上下左右四个维度上的寻找。
        2. 先根据源与目的地卫星的经纬度添加约束条件，即为搜索空间降低为两个维度，降低算法时间复杂度
          1. 问题变为：从球形矩阵源点到汇点的有向图路径问题，（类似于曼哈顿街网络）,即，变成有向无环图（DAG），DAGShort算法基于拓扑排序，时间复杂度为O(V+E)
        3. A*算法，一种启发式算法，代价函数为f(n) = g(n) + h(n), h(n): 启发评估函数，表示当前节点到最终节点的代价，可以指引搜索算法靠近终点，采用曼哈顿距离表示即，h(n) = ｜x1 - x2| + |y1 - y2|, (x1, y1) to (x2, y2). g(n) 路径权重函数（isl 链路权重）–>Dirstra shortest path,

    4. 针对不同qos等级，对path路径集合中不同权重路径的选择概率

    5. 路径概率计算，获取路径的所有链路带宽、时延、丢包率、跳数信息。根据这些信息判断是否该条路径满足所需要的qos要求，选择最能满足该流的路径下发流表。

    6. 引出问题，为当前具有不同 QoS 的流量选择合适的路径。但是，这可能会导致当前的低优先级流量占用后续高优先级流量的带宽。当后续高优先级流量到来，如果不可以找到满足起要求的路径，对路径上低优先级流量进行降级，同等优先级流量之间满足最大最小公平原则。
